# 医疗影像辅助诊断系统项目说明文档

## 1. 项目概述

本项目是一个基于深度学习的皮肤病医学影像辅助诊断系统，面向 HAM10000 皮肤病图像分类任务。系统使用 ECA-ResNet50 作为核心模型，结合 Streamlit 前端、Grad-CAM 可解释热力图、临床风险分层、医生反馈记录和反馈增量训练，形成从图像上传、AI 预测、结果解释、医生反馈到模型再训练的闭环流程。

系统支持 7 类皮肤病变识别：`nv`、`mel`、`bkl`、`bcc`、`akiec`、`vasc`、`df`。其中 `mel`、`bcc`、`akiec` 被视为恶性或高风险相关类别，用于辅助风险分层。

## 2. 核心功能

1. 上传皮肤病灶图片并进行 AI 分类。
2. 输出疾病类别、置信度、恶性风险概率和不确定性指标。
3. 生成 Grad-CAM 热力图，展示模型关注区域。
4. 提供高召回、平衡、稳健等临床辅助决策模式。
5. 支持医生反馈、病例记录、审核和微调候选样本导出。
6. 支持反馈错题增强、训练集难例挖掘和原始样本混合训练。
7. 支持 Docker 部署。

## 3. 当前模型版本

模型权重统一放在 `outputs/checkpoints/` 目录下。前端当前只保留两个可选模型：

| 模型选项 | 权重文件 | 定位 |
|---|---|---|
| `v8（强化训练最优）` | `eca_resnet50_v8_best_acc87_97.pth` | 当前默认前端模型，作为强化训练后的最优模型使用。 |
| `v2_finetuned（反馈增量）` | `eca_resnet50_v2_finetuned.pth` | 反馈训练保存的增量模型，用于展示反馈闭环训练结果。 |

`v2` 已不再作为前端可选项展示。当前已知主要指标如下：

| 模型 | Accuracy | Recall | F1 |
|---|---:|---:|---:|
| Baseline | 56.27% | 17.63% | 13.82% |
| ECA-ResNet50 v2 | 87.02% | 76.65% | 77.39% |
| ECA-ResNet50 v8 | 87.97% | 81.96% | 79.91% |

## 4. 反馈闭环训练策略

反馈训练不是简单地只用少量反馈数据继续训练，而是采用更稳妥的混合策略：

1. 使用 `feedback_data1/` 作为反馈数据目录。
2. 使用 `eca_resnet50_v8_best_acc87_97.pth` 作为反馈训练基础模型。
3. 对医生反馈错题进行数据增强。
4. 从原始训练集中挖掘错分样本和低置信样本。
5. 将反馈错题、增强样本、难例样本与原始训练样本混合训练。
6. 按约 1:6 的比例插入原始训练样本，降低灾难性遗忘风险。
7. 训练后只有在 Accuracy 稳定，且 Recall 或 F1 有改善时才保存反馈模型。

向导师解释时可以说明：反馈闭环的目标不只是让总体 Accuracy 每次都升高，更重要的是提升模型对反馈错例、低置信难例和临床困难样本的适应能力，同时通过保存门槛避免模型退化。

## 5. 运行方式

本地运行：

```bash
streamlit run app.py
```

Docker 构建：

```bash
docker build -t med-ai4 .
```

Docker 运行：

```bash
docker run -p 8501:8501 med-ai4
```

推荐使用 Docker Compose 一键构建并启动：

```bash
docker compose up --build
```

Compose 会将本地 `feedback_data1/` 挂载到容器的 `/data/feedback_data`，因此医生反馈记录和 `model_metrics.json` 指标文件会持久化保存，不会因为删除容器而丢失。

如果不用 Compose，也建议手动挂载反馈目录：

```bash
docker run -p 8501:8501 -v E:/AI/med_ai4/feedback_data1:/data/feedback_data med-ai4
```

访问地址：

```text
http://localhost:8501
```

当前 Docker 镜像只打包前端实际可选的两个部署模型：`v8（强化训练最优）` 和 `v2_finetuned（反馈增量）`。历史 `v2` 权重不再打入镜像，以减小镜像体积。

## 6. 交付建议

建议交付：

1. 核心源码：`app.py`、`med_*.py`。
2. 训练和评估脚本：`scripts/`。
3. 实验 Notebook：`Graduation_Project/01_data_process.ipynb`。
4. 模型权重：`outputs/checkpoints/` 中必要 `.pth` 文件。
5. 指标和图表：`outputs/metrics/`、`outputs/figures/`。
6. 部署文件：`requirements.txt`、`Dockerfile`、`docker-compose.yml`、`.dockerignore`。
7. 文档：`项目说明文档.md`、`各文件说明文档.md`。

不建议交付：

1. `__pycache__/`。
2. `scripts/__pycache__/`。
3. `outputs/pycache_check/`。
4. `*.pyc`。
5. `med_ai_pro/` 虚拟环境。
6. `.git/`、`.idea/`、`.vscode/`。
