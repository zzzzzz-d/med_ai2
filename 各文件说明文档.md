# 各文件说明文档

本文档说明项目中需要关注和交付的主要文件，不包含 `__pycache__`、`.pyc`、`outputs/pycache_check/`、虚拟环境缓存等临时文件。

## 1. 根目录核心文件

| 文件 | 作用 |
|---|---|
| `app.py` | Streamlit 前端入口，负责页面展示、模型选择、图片上传、推理、Grad-CAM 展示和临床工作台入口。 |
| `med_config.py` | 全局配置，管理模型路径、反馈目录、随机种子、风险阈值和模型注册表。当前前端只注册 `v8` 和 `finetuned`。 |
| `med_model_arch.py` | 定义 ECA-ResNet50 模型结构和兼容权重加载函数。 |
| `med_models.py` | 加载模型和 Grad-CAM 对象，并使用 Streamlit 缓存减少重复加载。 |
| `med_inference.py` | 提供温度缩放、不确定性计算、风险分层和随访建议生成逻辑。 |
| `med_report.py` | 生成 HTML 报告内容和图片 Base64 编码。 |
| `med_feedback.py` | 管理医生反馈、病例历史、CSV 读写、Schema 兼容和微调候选样本导出。 |
| `med_clinical_ui.py` | 渲染医生反馈、审核、病例历史和错题分析等临床工作台模块。 |
| `med_metric_store.py` | 保存和读取模型指标，用于前端展示模型表现。 |
| `requirements.txt` | Python 依赖列表。 |
| `Dockerfile` | Docker 部署配置，复制核心源码和部署模型，启动 Streamlit 服务。 |
| `docker-compose.yml` | Docker Compose 启动配置，一键构建并启动服务，同时挂载 `feedback_data1/` 实现反馈数据和指标持久化。 |
| `.dockerignore` | Docker 构建忽略规则，排除数据集、虚拟环境、缓存、训练文件和大部分实验输出。 |
| `.gitignore` | Git 忽略规则。 |

## 2. Notebook

| 文件 | 作用 |
|---|---|
| `Graduation_Project/01_data_process.ipynb` | 主要实验 Notebook，包含数据处理、训练、评估、模型保存和反馈闭环训练代码。 |

Notebook 中的反馈训练已调整为：

1. 使用 `feedback_data1/`。
2. 使用 v8 强化训练最优模型作为反馈训练起点。
3. 自动生成扩展反馈训练池。
4. 混合反馈错题、增强样本、训练集难例和原始训练样本。
5. 通过保存门槛避免保存退化模型。

## 3. scripts 目录

| 文件 | 作用 |
|---|---|
| `scripts/train_baseline_resnet50.py` | 训练 Baseline ResNet50。 |
| `scripts/export_table_5_1_metrics.py` | 导出模型 Accuracy、Precision、Recall、F1 等评估指标。 |
| `scripts/generate_comparison_figures.py` | 生成论文图表，例如混淆矩阵和 Recall 对比图。 |
| `scripts/finetune_from_feedback.py` | 根据医生反馈样本进行增量微调。 |
| `scripts/expand_feedback_training_pool.py` | 执行反馈错题增强和训练集难例挖掘，生成扩展训练池。 |

## 4. outputs 目录

| 路径 | 作用 |
|---|---|
| `outputs/checkpoints/` | 保存模型权重。 |
| `outputs/metrics/` | 保存模型评估指标 CSV。 |
| `outputs/figures/` | 保存论文图表和实验可视化结果。 |

主要权重文件：

| 文件 | 作用 |
|---|---|
| `eca_resnet50_v8_best_acc87_97.pth` | 前端默认模型，v8 强化训练最优模型。 |
| `eca_resnet50_v2_finetuned.pth` | 反馈增量模型。 |
| `eca_resnet50_v2_best.pth` | 历史 v2 模型，目前不在前端展示。 |
| `baseline_resnet50_best*.pth` | Baseline 对照模型。 |
| `eca_resnet50_v9_best_acc86_82.pth` | 后续实验模型，可作为实验记录保留。 |

## 5. feedback_data1 目录

| 文件或目录 | 作用 |
|---|---|
| `clinical_feedback_log.csv` | 医生反馈主日志。 |
| `case_history_log.csv` | 病例历史记录。 |
| `finetune_candidates.csv` | 审核通过的原始反馈训练候选样本。 |
| `finetune_candidates_expanded.csv` | 反馈增强和训练集难例挖掘后的扩展训练池。 |

## 6. Docker 部署文件

Docker 镜像会复制：

1. 核心运行源码。
2. `requirements.txt`。
3. `outputs/checkpoints/eca_resnet50_v2_finetuned.pth`。
4. `outputs/checkpoints/eca_resnet50_v8_best_acc87_97.pth`。

Docker 镜像当前不再复制历史 `eca_resnet50_v2_best.pth`，因为前端已经不显示 v2 模型，只保留 v8 和反馈增量模型。

推荐启动方式：

```bash
docker compose up --build
```

`docker-compose.yml` 会把本地 `feedback_data1/` 挂载到容器 `/data/feedback_data`，用于持久化医生反馈 CSV、病例历史和 `model_metrics.json` 指标文件。

Docker 镜像不会复制：

1. `dataset/`。
2. `med_ai_pro/`。
3. `feedback_data1/`。
4. `Graduation_Project/`。
5. `scripts/`。
6. `__pycache__/`。
7. `*.pyc`。
